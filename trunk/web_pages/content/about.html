About the OpenRecord project

The goal of the project is to create an open source software product. The product we're making is a wiki engine, but where most wiki engines are geared toward pages of text, OpenRecord is geared toward loosely-structured database content.

History

  • February 2005 -- started writing code
  • August 2005 -- first demo documents, and first real demos
  • November 2005 -- became a project of the Dojo Foundation
  • February 2006 -- first screencasts
  • 2006 -- pick a server side datastore to use
  • 2006 -- convert code to use more of the Dojo toolkit

Status (as of April 2006)

We currently have two programmers working on OpenRecord part time, and we're looking for more volunteers. One of the main things we're doing now is looking at different server-side datastores. Later this year we hope to commit to a single datastore or file format, so people will be able to start actually saving their data in OpenRecord.

License

All of the code we're writing is available under open source licenses.
